# QLoRA Fine-Tuning Analysis

## Overview

Three QLoRA training approaches were tested on **Qwen 2.5 7B** to determine
whether a small, locally-runnable model can close the gap with commercial
models on Ontario-specific insurance questions.

**Best result: 65.68%** (+5.9pp over base Qwen 59.76%) — the highest
open-source accuracy on the leaderboard.

## Training Setup

- **Hardware:** Apple M3 Pro, 36 GB unified memory
- **Framework:** MLX QLoRA (Apple Silicon native)
- **Base model:** Qwen 2.5 7B Instruct (4.4 GB GGUF via Ollama)
- **LoRA config:** 8 layers, rank 8, alpha 16
- **Trainable parameters:** 5.7M (0.076% of 7.6B)
- **Training:** lr=2e-5, 200 iterations, batch size 4
- **Validation loss:** 1.49 → 0.61 (no overfitting observed)

## Approach 1: Answer-Only Labels (Failed)

**Hypothesis:** The model can learn the Q→A mapping from correct answer labels alone.

**Data:** 386 (question, correct_answer_letter) pairs from the training MCQ pool.

**Result:** No improvement (~60%). The model cannot internalize regulatory
knowledge from answer labels without reasoning traces.

**Why it failed:** MCQ answer letters are arbitrary tokens. Without reasoning
context, there's nothing for the model to generalize from. This is equivalent
to memorization, and 386 examples is insufficient for memorization of a 
domain this large.

## Approach 2: Synthetic Reasoning Templates (Failed)

**Hypothesis:** GPT-generated chain-of-thought reasoning traces provide 
the missing reasoning context.

**Data:** 386 (question, synthetic_CoT, answer) triples. CoT generated by
prompting a stronger model to explain each answer step-by-step.

**Result:** Degraded to **47.9%** — worse than the base model (59.76%).

**Why it failed:** The synthetic traces contain subtle distribution mismatches:
- Template language patterns ("Let's think step by step...") that the model
  overfits to as surface features
- Reasoning that sounds correct but references knowledge the base model 
  doesn't have, creating a distribution gap between training and inference
- At 386 examples, the model memorizes template artifacts rather than 
  learning domain reasoning

## Approach 3: Filtered Self-Distillation (Success)

**Hypothesis:** If we only train on reasoning traces where the model itself
arrived at the correct answer, we teach it to replicate its own best patterns.

**Process:**
1. Run base Qwen 2.5 7B on all 386 training questions with CoT prompting
2. Check which answers match the ground truth
3. Keep only the 253 traces (65.5%) where the model was correct
4. Train QLoRA on these filtered traces

**Result:** **65.68%** (+5.9pp over base)

**Why it works:**
- The reasoning traces are *in-distribution* — they use the model's own
  vocabulary, phrasing, and reasoning patterns
- Filtering removes traces where the model hallucinated or guessed correctly
  for wrong reasons
- The model learns to be more consistent at applying reasoning patterns it
  already sometimes uses
- 253 high-quality traces > 386 noisy traces

## Also Tested: Self-Consistency Voting (Failed)

**Hypothesis:** Running the base model 5 times at temperature 0.7 and
taking the majority vote improves accuracy.

**Result:** No improvement (59.76%). When the model lacks domain knowledge,
sampling different reasoning paths doesn't help — it just samples different
wrong answers.

**Insight:** Temperature-based voting helps when the model has the knowledge
but sometimes fails to access it. For this task, the knowledge deficit is
too large for sampling to overcome.

## Summary

| Approach | Data | Accuracy | Delta |
|:---------|:-----|:---------|:------|
| Base Qwen 2.5 7B | — | 59.76% | — |
| + Answer-only QLoRA | 386 pairs | ~60% | +0pp |
| + Synthetic CoT QLoRA | 386 triples | 47.9% | -11.9pp |
| + **Filtered self-distill QLoRA** | **253 traces** | **65.68%** | **+5.9pp** |
| + Self-consistency (5x, t=0.7) | — | 59.76% | +0pp |

## Key Takeaway

**Data quality > data quantity.** 253 in-distribution self-generated traces
outperform 386 synthetic traces by 18pp. The model learns best from its own
correct reasoning patterns, not from external templates that look right but
are distributionally mismatched.

## Reproducing

```bash
# Generate reasoning traces
make eval CONFIG=configs/v9_qlora_generate_traces.yaml

# Filter to correct traces only
python scripts/filter_traces.py --min-accuracy 1.0

# Train QLoRA
python scripts/train_qlora_mlx.py \
    --model qwen2.5:7b-instruct \
    --data data/filtered_traces.jsonl \
    --layers 8 --rank 8 --alpha 16 \
    --lr 2e-5 --iters 200

# Evaluate
make eval CONFIG=configs/v9_qlora_v3_filtered.yaml
```

Adapter weights are stored in `adapters/qlora_v3_filtered/`.
